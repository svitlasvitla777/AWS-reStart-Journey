# AWS Cloud Quest — Cloud Practitioner

During the AWS re/Start programme I worked through AWS Cloud Quest — a gamified learning environment where you solve real challenges and learn how AWS services work in practice. I liked this format because it forces you to actually do things instead of only reading theory.

## What I Practiced

Working through the Cloud Quest scenarios helped me get hands-on experience with:

- Core cloud concepts and how they apply in practice
- Launching and managing AWS resources
- Understanding how different AWS services connect to each other
- Troubleshooting when something does not work as expected

---

## AWS Services Used

| Service | Purpose |
|---|---|
| Amazon EC2 | Launching and managing virtual servers |
| Amazon S3 | Object storage and static website hosting |
| AWS IAM | Managing users, roles, and access policies |
| Amazon VPC | Networking and security configuration |
| AWS CloudWatch | Monitoring resources and setting up alerts |

## Personal Note

What I find interesting is how many cloud topics connect directly to things I have seen in business. When learning about IAM and security policies I immediately think about governance and control frameworks from finance. When AWS talks about cost management, my finance background kicks in and I start thinking about budgets, cost allocation, and optimisation.

For me this is not only about learning technology. It is about understanding the infrastructure that powers modern companies — and being able to work at that intersection of technical and business thinking.
